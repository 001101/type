Scale formula

The formula for our scale is new to provide all the needs stated above.

The scale assumes y0 is 12px.

yn = yn-1 + ⌊(n-2) ÷ 4⌋ · 2

Note: When using IBM Plex Serif, you always subtract the expected font size by 1px.

Styles

Typography creates texture into our design. It guides viewers to read and understand the hierarchy of content. We need to manage what we are delivering through typographic treatment with controlled usage of type styles and weights.

Weights

We suggest to use IBM Plex Light, Regular, SemiBold, and Italic for digital experiences. Note that IBM Plex Sans Bold, ExtraLight, Text, and Thin are not for digital experiences.

Line Height

Line heights have been determined based on type sizes.

Line Length

For a better reading experience, IBM Type’s code automatically sets a max-width for text length based on the type style and IBM Grid column width. According to The Elements of Typographc Style, 45 to 75 characters is widely regarded as a satisfactory length of line for a single-column page set in a serifed text face in a text size.

Alignments

Elements of good typography include consistency, hierarchy, and alignment. Flushing left creates strong alignments for the eye to follow, aiding readability. It is our standard for text layout.

Bottom Margin

We set up bottom margin based on the type style used. Use increments of 4px (0.25rem) to define custom bottom margins for specific text parings.

Fluid Type

To create a better experience along with our fluid IBM Grid, fluid type is the way to go. We set up our fixed type scales at the breakpoints (320px, 640px, 1056px, and 1650px) and have them be fluid between. This helps us maintain the quality of typography.

Why fluid type?

It potentially creates better rags when you scale up the screen.

Type Color

To maintain the quality and recognition of our work, we always consider how to use our color in design. Legibility and accessibility or the two keys.

Keep our type color neutral when it is shown in paragraphs. We never use pure black (#000) for text. Instead, we use a range of grays. We use blue for primary actions. More color guidance will come soon.
