Style

Typography creates texture. It guides users to read and understand the hierarchy of information. Typographic treatment and the controlled usage of types styles helps manage the display of content.

Weights

We suggest to use IBM Plex Light, Regular, SemiBold, and Italic for digital experiences. Note that IBM Plex Sans Bold, ExtraLight, Text, and Thin are not for digital experiences.

Oops!

Bold, Medium, Text, Extralight and Thin should not be used in websites.

Italic

Each weight has an Italic style, which should be used sparingly. Use italic when you need to emphasize certain word in a sentence. Here are some cases you use italic: titles of works, technical terms, names of devices, captions.

Case

Write content in sentence case. Title case can be used in proper titles, product names, and serive names. but avoid uppercase or “all caps.”

Oops!

Uppercase is not to be used for writing. It yells too much and is hard to read.

Line length

Max width for text length is based on the column width, type scales and the limit of characters. They are built in our type code. Use the standard—45 to 75 characters—for the line length of a single-column page set in a text size.

Alignments

Text alignments organize content for the eye to follow, aiding readability. We use this standard for Western Latin languages  to create clean and consistent layouts.

Oops!

Centered type does not fit the IBM typography approach, which creates relationships based on strong alignments and grid column systems.

Oops!

Flush right is reserved for languages that read right to left but is hard to read in Western Latin languages.

Oops!

Justified copy can fit into IBM typography, but we recommend not using this approach as it is difficult to do well and requires strong typographic expertise.

Margin Bottom

We set up bottom margin based on the type scales and weights. Use the increments of 4px (0.25rem) to define the bottom-margin for text pairings. They are built in our type code.

Bottom Margin

Fluid Type

Use fluid type to create a unified experience with the grid. The fixed type scales are fluid between breakpoints to maintain the quality of typography.

Why fluid type?

It creates cleaner type rag when you scale up the screen and provides a better line-length for reading.

Type Color

Careful consideration of color maintains quality and recognition for users. Legibility and accessibility are two keys. Keep colored type neutral when in paragraphs. Never use pure black (Hex value: #000000) for text, but instead, a range of grays. Use primary blue for primary actions.

Interactive text

Primary colors are used for text links or main actions.

Color type for specific usage

Other use cases for colored type: code snippet, warning, alert, etc.
